# Руководство по продукту

## 1. Что это за приложение

StudioVD — это рабочее место для студии и исполнителей: заказы, этапы, файлы и чат в одном месте. Цель — чтобы заказ не терялся в переписках и мессенджерах, а статус и договорённости были видны всем участникам.

## 2. Как устроена система

### Роли

- **Админ (студия)** — создаёт и ведёт заказы, назначает исполнителей, общается в чате, согласует этапы.
- **Исполнитель** — видит назначенные заказы, работает по этапам, пишет в чате и загружает результаты.

Доступ к данным ограничен ролью и участием в заказе: исполнитель не видит чужие заказы.

## 3. Как пользоваться

### Создание заказа

Заказ создаётся из админки (поток «быстрый заказ» или список заказов — в зависимости от раздела в приложении). После создания заказ получает статус и может быть привязан к исполнителю.

### Назначение

Студия назначает исполнителя на заказ. После назначения исполнитель видит заказ в своём списке и может открыть карточку, чат и этапы.

### Чат

Чат привязан к заказу: переписка только между студией и исполнителем по этому заказу. В чате есть статусы «в сети» / «был недавно», индикатор печати, ответы на сообщения и копирование текста. **Enter** отправляет сообщение, **Shift+Enter** — новая строка.

### Этапы

Этапы (чекпоинты) отражают ход работы: согласование, правки, сдача. Их прохождение видно в карточке заказа.

## 4. Уведомления

Уведомления создаются на сервере и попадают в таблицу `notifications`. Клиент подписан на изменения в реальном времени: при новой записи обновляется список и счётчики (бейджи в навигации).

Пользователь может открыть колокольчик, перейти по ссылке из уведомления и отметить прочитанным. При включённых интеграциях возможны дополнительные каналы (push, Telegram) — в зависимости от настроек проекта.

## 5. Мобильное использование

На телефоне доступна нижняя навигация с учётом safe-area (чёлка и домашние индикаторы). Чат можно открыть в виде плавающей кнопки (FAB) с учётом отступа над панелью навигации. Рекомендуемые зоны нажатия — не меньше 44×44 px.

## 6. Горячие клавиши (чат)

| Действие        | Клавиши        |
|-----------------|----------------|
| Отправить       | Enter          |
| Новая строка    | Shift+Enter    |

## 7. FAQ

**Почему не вижу сообщение сразу?**  
Нужны переменные окружения Supabase на клиенте и включённая репликация Realtime для таблицы `messages`.

**Почему счётчик уведомлений мигал?**  
Счётчик синхронизируется с сервером; при появлении нового уведомления UI может сначала обновиться оптимистично, затем подтвердить данные с API.

**Как откатить ответ на сообщение перед отправкой?**  
Нажмите «✕» в строке с превью ответа над полем ввода.
